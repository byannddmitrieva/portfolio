# Portfolio Development Guidelines

## Project Context

This is the premium personal portfolio of designer Anna Dmitrieva. The visual style is minimal, editorial, and elegant. Every change should feel intentional, refined, and polished.

## Design Principles

- Preserve the existing design language unless a redesign is explicitly requested.
- Maintain the established typography, spacing, layout, and animation behavior unless changes are specifically requested.
- Reuse the existing HTML structure and CSS classes whenever possible.
- Treat the mobile experience as a top priority and verify that every change works well across screen sizes.

## Technical Guidelines

- Do not introduce frameworks, libraries, or other dependencies.
- Keep the website fast and lightweight.
- Write semantic, clean, and accessible HTML.
- Keep CSS organized, reuse existing rules, and avoid duplication.
- Optimize assets and implementation choices for performance and accessibility.
- Avoid creating unnecessary files.

## Workflow

1. Before editing, briefly explain the proposed plan.
2. Make only the requested changes and avoid unrelated modifications.
3. After editing, show the diff and list every modified file.
4. Wait for explicit approval before creating a commit.
5. Wait for separate explicit approval before pushing to GitHub.

Never commit or push without explicit approval.
